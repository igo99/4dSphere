How a web page is built — a short primer

Written for someone who is curious but hasn't written code before. Uses the 4D sphere visualiser (index.html) as a running example.

HTML — structure CSS — style JavaScript — behaviour
The house analogy. Think of a web page as a house. HTML is the brickwork — it decides what rooms exist and what order they go in. CSS is the interior design — paint colours, furniture, lighting. JavaScript is the electricity — it makes things move, respond, and change. All three live in the same file and work together.
Part 1

HTML — Structure

HTML stands for HyperText Markup Language. It describes the contents of a page using elements, each wrapped in tags.

Tags

A tag is a word inside angle brackets. Most elements have an opening tag and a closing tag (with a slash). Everything between them is the element's content.

<p>This is a paragraph.</p>

<h1>This is a big heading.</h1>

<button>Click me</button>

Attributes

Tags can carry extra information called attributes — written inside the opening tag as name="value" pairs.

<input type="range" min="-180" max="180" value="0">

<!-- That renders as the slider below: -->
live result

Two attributes are especially important because CSS and JavaScript use them to target specific elements:

  • id="name" — a unique label for one element (like a name tag)
  • class="name" — a group label shared by many elements

Nesting

Elements live inside other elements, forming a tree. The outer element is called the parent; the inner ones are children.

<div id="panel">           <!-- parent -->

  <h1>4D Sphere</h1>       <!-- child -->

  <div class="ctrl">         <!-- child -->
    <input type="range">   <!-- grandchild -->
  </div>

</div>
This nesting structure is called the DOM — Document Object Model. JavaScript can walk this tree, find any element, and change it.

CSS — Style

CSS stands for Cascading Style Sheets. It controls how elements look — colours, sizes, spacing, fonts, layout.

The rule

A CSS rule has two parts: a selector (which element to style) and one or more declarations (what to do to it).

selector {
  property: value;
  another-property: another-value;
}

Where CSS lives

In index.html, all the CSS is inside a <style> block in the <head>. For example, the sidebar panel is styled like this:

#panel {
  position: fixed;        /* stays put while you scroll */
  top: 0; left: 0; bottom: 0;
  width: 210px;
  background: rgba(4,4,18,0.94); /* near-black, slightly transparent */
  backdrop-filter: blur(12px); /* frosted-glass effect */
}
Colours in CSS can be written as names (red), hex codes (#3a6fd8), or rgb()/rgba() — the fourth number in rgba is opacity (0 = invisible, 1 = solid).
Part 3

JavaScript — Behaviour

JavaScript (JS) is a programming language that runs inside the browser. It can read and change the page in response to events — clicks, slider moves, the passage of time.

Variables

A variable stores a value. Use const when the value won't change, let when it might.

const radius = 1.0;          // a number that won't change
let angle = 0;               // a number we'll update every frame
const name = "Hopf fiber";   // text (called a string)
const animating = true;      // a boolean: true or false

Functions

A function is a reusable block of code. You define it once, then call it by name whenever you need it.

// Define
function double(x) {
  return x * 2;
}

// Call
const result = double(5);  // result is now 10

In index.html, stereo() is a function that takes a 4D point and returns its 3D projection:

function stereo([x, y, z, w]) {
  const d = 1 - w;         // distance from north pole
  return [x/d, y/d, z/d]; // the projected 3D point
}

Objects

An object groups related values together under one name, using { key: value } syntax.

const S = {
  nFibers: 64,      // how many fibers to draw
  speed: 0.4,       // animation speed
  animating: true,  // is it running?
};

// Read a value:
console.log(S.speed);  // prints 0.4

// Change a value:
S.speed = 1.0;

Events

The browser fires events when the user interacts — clicks, key presses, slider moves. You attach a function (called a listener) that runs when the event happens.

const btn = document.getElementById('bPause');

btn.addEventListener('click', () => {
  S.animating = !S.animating;  // flip true↔false
});

The animation loop

To make things move smoothly, JavaScript has requestAnimationFrame — it calls your function ~60 times per second, in sync with the screen.

function loop() {
  requestAnimationFrame(loop); // schedule next call
  angle += 0.01;               // advance the angle
  redraw();                    // update what's on screen
}

loop(); // start it

This is exactly how index.html works — the render loop at the bottom of the script runs every frame, advances S.animAngle, and calls redrawFibers() to update the positions.

Part 4

All three together

Let's trace what happens the moment you move the Speed slider in index.html.

LayerWhat it does
HTML The <input type="range" id="sSp"> element exists in the page. The browser draws it as a slider automatically.
CSS The rule input[type=range] styles it: 2px tall, dark track, a small blue thumb. The --p CSS variable is updated to fill the track up to the current value.
JavaScript bindRange('sSp', 'vSp', 'speed', S, false, false) has already attached an input listener to that slider. When you drag it, the listener fires, reads the new value, and writes it to S.speed.
JavaScript (loop) The render loop is running every frame. It reads S.speed to calculate how far to advance S.animAngle this frame. Since you changed S.speed, the rotation immediately speeds up or slows down.

All four things happen in under a millisecond. From your perspective: you dragged a slider and the animation changed speed.

Reference

Key pieces of index.html

The importmap — loading Three.js

Three.js is a 3D graphics library. Instead of downloading it, the page loads it from the internet using a module import. The <script type="importmap"> tells the browser where to find it:

<script type="importmap">
{
  "imports": {
    "three": "https://cdn.jsdelivr.net/npm/three@0.160.0/..."
  }
}
</script>

<script type="module">
import * as THREE from 'three'; // now THREE.* is available
</script>

The state objects — S and CS

S holds everything about Hopf mode; CS holds everything about slice mode. Keeping all mutable values in one place means any part of the code can read or change them, and you always know where to look.

rebuildFibers vs redrawFibers

Creating geometry (allocating memory) is slow. Updating existing geometry is fast. So the code separates these two operations:

  • rebuildFibers() — called only when you change the number of fibers or the segment count. Creates fresh objects.
  • redrawFibers() — called every animated frame. Just overwrites the position numbers already in memory. No allocation.

updateSlice()

For the cross-section mode, the sphere geometry never changes — it's always a unit sphere. Instead, sliceMesh.scale.setScalar(r) shrinks or grows it by the right radius each frame. Scaling an existing mesh is even cheaper than rewriting its positions.

The math in plain English

The four-dimensional sphere S³ contains points that are too far away to see directly. The visualiser uses stereographic projection to squash 4D down to 3D: imagine shining a light from the top of S³ onto a flat 3D screen below — every point casts a shadow, and the shadow is what Three.js draws.

Rotating in 4D works the same way as rotating in 3D — you pick a plane to rotate in (in 3D you rotate in the xy-plane; in 4D you can also rotate in xw, yw, or zw). The sliders control which 4D plane is spinning.
